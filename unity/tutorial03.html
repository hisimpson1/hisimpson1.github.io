2D PPU

유니티 2D 게임에서 텍스쳐 PPU와 씬뷰의 격자 한칸의 관계는?              



유니티(Unity)에서 씬 뷰에 보이는 저 **격자(Grid) 한 칸은 기본적으로 '1 유니티 단위(1 Unity Unit)'**를 의미합니다.

결론부터 말씀드리면, "한 칸이 곧 PPU"인 것은 아니지만, PPU 설정에 따라 스프라이트가 저 한 칸에 어떻게 들어가는지가 결정됩니다.

이 개념을 쉽게 이해하실 수 있도록 핵심 내용을 정리해 드릴게요.

1. Unity Unit vs PPU (Pixels Per Unit)

  • Unity Unit (격자 한 칸): 유니티 월드 내의 거리 단위입니다. 물리 엔진이나 좌표계에서 기본이 되는 1.0의 크기입니다.

  • PPU (Pixels Per Unit): 스프라이트 이미지의 **'몇 픽셀을 유니티 1단위(한 칸)로 취급할 것인가'**를 결정하는 설정값입니다.

2. 예시로 이해하기

만약 내가 가져온 캐릭터 이미지의 해상도가 64x64 픽셀이라고 가정해 보겠습니다.

스프라이트 PPU 설정결과 (씬 뷰에서의 크기)
PPU = 64이미지가 격자 **정확히 한 칸(1 unit)**에 꽉 차게 들어갑니다.
PPU = 32이미지가 격자 **두 칸(2 units)**의 크기를 차지하게 됩니다.
PPU = 100 (기본값)이미지가 격자 한 칸보다 작게 보입니다. (약 0.64칸)

3. 왜 이게 중요한가요?

보통 2D 픽셀 아트 게임을 만들 때 PPU 설정이 매우 중요합니다.

  • 일관성: 모든 스프라이트의 PPU를 동일하게 맞춰야 캐릭터와 배경의 해상도 이질감이 없습니다.

  • Tilemap 작업: 타일셋이 16x16 픽셀이라면, PPU를 16으로 설정해야 타일 하나가 격자 한 칸에 딱 맞게 들어갑니다.

팁: 만약 픽셀 게임을 만드신다면, 사용 중인 타일의 가로 픽셀 크기에 맞춰 스프라이트의 Pixels Per Unit 값을 수정해 보세요!

혹시 지금 작업 중인 아트셋의 픽셀 사이즈를 알려주시면, 적절한 PPU 값을 추천해 드릴까요?


2D 직교 좌표계로 설정한 상태에서는?              

2D 직교 좌표계(Orthographic) 모드에서도 격자 한 칸은 여전히 1 Unity Unit입니다. 하지만 화면에 보이는 방식이 3D와는 조금 다릅니다.

직교 좌표계 환경에서 PPU와 카메라의 관계를 이해하면 작업이 훨씬 수월해집니다.

1. 격자 한 칸의 의미

직교 좌표계에서 Camera의 Size 속성을 보면 이 격자의 개념이 명확해집니다.

  • Camera Size가 5일 경우: 화면의 정중앙에서 위쪽 끝까지 격자 5칸이 들어갑니다. (즉, 화면 전체 세로 높이는 10단위가 됩니다.)

  • 이때 PPU가 100이라면, 화면 세로 전체에 픽셀이 총 $10 X 100 = 1000픽셀만큼 그려진다는 뜻입니다.

2. '픽셀 퍼펙트(Pixel Perfect)'를 원하신다면?

만약 픽셀 아트 게임을 만들고 계셔서 스프라이트가 격자에 딱딱 맞기를 원하신다면 다음 공식을 기억하세요.

PPU = 타일의 픽셀 크기

  • 예를 들어, 타일 하나가 16x16 픽셀이라면 이미지 설정에서 PPU를 16으로 바꿔주세요.

  • 그러면 스프라이트 하나가 정확히 **격자 한 칸(1 unit)**을 차지하게 되어 배치하기가 매우 편해집니다.

3. 직교 좌표계 작업 팁

  • Snap 기능: Ctrl을 누른 채로 오브젝트를 움직이면 격자 단위(1 unit)로 딱딱 끊어서 이동할 수 있습니다.

  • Pivot(중심점): PPU를 맞춰도 격자에 안 맞는다면, 스프라이트 설정에서 Pivot이 Center로 되어 있는지 확인해 보세요. 격자 선에 맞추고 싶다면 Top Left나 Bottom Left로 변경하기도 합니다.
